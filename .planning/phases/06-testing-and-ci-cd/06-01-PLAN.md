**CI Workflow (.github/workflows/ci.yml):**

Create a GitHub Actions workflow with TWO jobs, heavily annotated with CCA exam explanations:

**Triggers (on: block):**
- `on: push` to main (runs test-and-lint)
- `on: pull_request` (runs claude-review)
- `on: schedule` with `cron: '0 6 * * *'` (nightly at 6 AM UTC -- runs test-and-lint). Add inline comment: `# schedule/cron: CICD-02 requires nightly runs for regression detection (CCA exam: cron ensures test suite runs even without commits)`

Job 1 -- `test-and-lint` (on push to main AND nightly cron):
- Uses `ubuntu-latest`, Python 3.13, Poetry
- Steps: checkout, setup-python, install Poetry, `poetry install`, `poetry run pytest`, `poetry run ruff check src/`
- Standard CI -- no Claude involvement

Job 2 -- `claude-review` (on pull_request only):
- Uses `ubuntu-latest`
- Runs `claude -p --bare --output-format json --allowedTools Read,Grep,Glob` with a prompt that reviews the PR diff for CCA compliance
- Captures output to a variable (e.g., `REVIEW_JSON`)
- **CRITICAL: Parse the JSON envelope before posting.** `--output-format json` produces a JSON object, not plain text. Extract the human-readable result using `jq -r '.result'` (or `.content` depending on envelope shape). Add inline comment: `# jq -r '.result': --output-format json returns a JSON envelope; posting raw JSON as a PR comment is unreadable (CCA teaching point: structured output requires programmatic parsing)`
- Posts the EXTRACTED text (not raw JSON) as a PR comment using `gh pr comment --body "$REVIEW_TEXT"`
- Uses `${{ secrets.ANTHROPIC_API_KEY }}` for authentication

Every CCA-relevant flag MUST have an inline YAML comment explaining its exam significance:
- `# -p flag: MANDATORY for non-interactive CI (CCA exam: without -p, pipeline hangs waiting for input)`
- `# --bare flag: RECOMMENDED for reproducibility (no CLAUDE.md loaded, deterministic behavior)`
- `# --output-format json: structured output at CLI level (CCA exam: enables programmatic parsing)`
- `# --allowedTools Read,Grep,Glob: sandboxing (CCA exam: principle of least privilege for CI agents)`
- `# ANTHROPIC_API_KEY as secret: credentials in user-level config, never project files (CCA rule)`

The workflow YAML is itself a CCA teaching artifact per user decision.
